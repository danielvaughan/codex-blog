# SEO PR 2 — Answer Engine Discoverability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make codex.danielvaughan.com explicitly discoverable by AI answer engines (ChatGPT search, Perplexity, Claude, Google AI Overviews, Bing Copilot) by adding: an AI-crawler allowlist in `robots.txt`, an `/llms.txt` curation file at site root, IndexNow infrastructure for rapid Bing+ChatGPT indexing, and a verification-meta-tag scaffold so Google Search Console / Bing Webmaster can be plugged in when the user has their codes.

**Architecture:** Four independent deliverables, each a small-to-medium file change, plus a GitHub Actions workflow that fires an IndexNow ping whenever posts change. No code plumbing ties them together — each can be added, tested, reverted independently. Everything remains compatible with GitHub Pages legacy builder (no custom Jekyll plugins required); the workflow runs separately from the Pages build and only affects the IndexNow/Bing ping, not the published site.

**Tech Stack:** Jekyll 3 (GitHub Pages legacy), Liquid, GitHub Actions (new), curl + bash for the IndexNow pinger.

**Testing note:** Same as PR 1 — "tests" mean build + grep + local serve + browser check. No unit test framework. Each task ends with an explicit verification step before committing.

## Task 2: llms.txt curated index at site root

**Files:**
- Create: `/Users/danielvaughan/Development/git/codex-blog/llms.txt`

**Context:** `llms.txt` is an emerging convention (2024-2026) for a Markdown-formatted site index optimised for LLM ingestion. Adoption is unproven as a citation ranking signal, but cost to add is near-zero and several major sites (Anthropic, Cloudflare, Mintlify, Cursor) ship one. The spec is: H1 site name, a blockquote summary, then sectioned annotated Markdown link lists.

We'll create it as a **Jekyll page** (not a static file) so it renders site metadata dynamically from `_config.yml` and auto-includes links to the RSS feed + sitemap + tags page without hand-maintenance. It lives at site root with a `permalink: /llms.txt` and a forced `layout: null` so Jekyll doesn't wrap it in HTML.

## Task 4: IndexNow ping workflow (GitHub Actions)

**Files:**
- Create: `.github/workflows/indexnow.yml`

**Context:** IndexNow infrastructure is now in place (key file + config). This task adds a GitHub Actions workflow that, on every push to main that touches files in `_posts/`, computes the URLs of added or modified posts and POSTs them to the IndexNow API (`https://api.indexnow.org/indexnow`). The workflow reads the key from `_config.yml` via a Python one-liner to avoid hardcoding it in the workflow file.

- [ ] **Step 1: Create the workflow directory**

```bash
cd /Users/danielvaughan/Development/git/codex-blog
mkdir -p .github/workflows
```

- [ ] **Step 2: Create the workflow file**

Use Write tool to create `.github/workflows/indexnow.yml` with this EXACT content:

```yaml
name: IndexNow Ping

# Fires on every push to main that touches _posts/. Computes the public URLs
# of added/modified post files and submits them to the IndexNow API so
# Bing (and indirectly ChatGPT search) re-crawl them within hours instead
# of days. Google ignores IndexNow — that path still relies on sitemap.xml.

on:
  push:
    branches: [main]
    paths:
      - '_posts/**'
  workflow_dispatch: {}

jobs:
  ping:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout with history
        uses: actions/checkout@v4
        with:
          fetch-depth: 2

      - name: Read IndexNow key from _config.yml
        id: key
        run: |
          KEY=$(python3 -c "
          import re, sys
          with open('_config.yml') as f:
              content = f.read()
          m = re.search(r'^indexnow_key:\s*\"?([a-fA-F0-9]+)\"?', content, re.MULTILINE)
          print(m.group(1) if m else '', end='')
          ")
          if [ -z "$KEY" ]; then
            echo "ERROR: indexnow_key not found in _config.yml"
            exit 1
          fi
          echo "key=$KEY" >> "$GITHUB_OUTPUT"

      - name: Compute changed post URLs
        id: urls
        run: |
          set -euo pipefail
          CHANGED=$(git diff-tree --name-only --diff-filter=AM -r HEAD^ HEAD -- _posts/ | grep -E '\.md$' || true)
          if [ -z "$CHANGED" ]; then
            echo "No post changes in this push; nothing to ping."
            echo "count=0" >> "$GITHUB_OUTPUT"
            exit 0
          fi
          URLS=""
          while IFS= read -r FILE; do
            BASE=$(basename "$FILE" .md)
            YEAR=$(echo "$BASE" | cut -d- -f1)
            MONTH=$(echo "$BASE" | cut -d- -f2)
            DAY=$(echo "$BASE" | cut -d- -f3)
            SLUG=$(echo "$BASE" | cut -d- -f4-)
            URL="https://codex.danielvaughan.com/${YEAR}/${MONTH}/${DAY}/${SLUG}/"
            URLS="${URLS}${URL}\n"
          done <<< "$CHANGED"
          echo "Computed URLs:"
          echo -e "$URLS"
          # Store as JSON array for the next step
          python3 -c "
          import json, sys
          urls = '''$(echo -e "$URLS")'''.strip().split('\n')
          urls = [u for u in urls if u]
          print(json.dumps(urls))
          " > /tmp/urls.json
          cat /tmp/urls.json
          COUNT=$(python3 -c "import json; print(len(json.load(open('/tmp/urls.json'))))")
          echo "count=$COUNT" >> "$GITHUB_OUTPUT"

      - name: POST to IndexNow
        if: steps.urls.outputs.count != '0'
        run: |
          set -euo pipefail
          KEY="${{ steps.key.outputs.key }}"
          PAYLOAD=$(python3 -c "
          import json
          urls = json.load(open('/tmp/urls.json'))
          body = {
              'host': 'codex.danielvaughan.com',
              'key': '$KEY',
              'keyLocation': f'https://codex.danielvaughan.com/{\"$KEY\"}.txt',
              'urlList': urls,
          }
          print(json.dumps(body))
          ")
          echo "Payload:"
          echo "$PAYLOAD"
          RESPONSE=$(curl -sS -w '\nHTTP %{http_code}' \
            -X POST \
            -H 'Content-Type: application/json; charset=utf-8' \
            --data "$PAYLOAD" \
            https://api.indexnow.org/indexnow)
          echo "Response: $RESPONSE"
          # IndexNow returns 200 (success), 202 (accepted), or 204 (no content).
          # Anything else is a failure.
          CODE=$(echo "$RESPONSE" | tail -1 | awk '{print $2}')
          case "$CODE" in
            200|202|204) echo "IndexNow accepted the ping (HTTP $CODE)" ;;
            *) echo "IndexNow rejected the ping (HTTP $CODE)"; exit 1 ;;
          esac
```
